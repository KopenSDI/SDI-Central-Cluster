## MC Criticality 재정의 규칙 (MALE Paper 기반)

```
┌────────────────────────────────────────────────────────────────────────────────┐
│                       Criticality Determination Rules                          │
│                       (MALE Paper Section IV 기반)                             │

우선순위 순서 (위에서 아래로 평가):

┌──────┬────────────────────────────────┬─────────────┬──────────────────────────┐
│ Rule │ Condition                      │ Criticality │ 사용 사례                │
├──────┼────────────────────────────────┼─────────────┼──────────────────────────┤
│  1   │ L >= 0.9                       │      C      │ 초저지연 (Boston Dynamics)│
├──────┼────────────────────────────────┼─────────────┼──────────────────────────┤
│  2   │ L >= 0.7 AND A >= 0.5          │      C      │ 안전 시스템              │
├──────┼────────────────────────────────┼─────────────┼──────────────────────────┤
│  3   │ A >= 0.6 AND L >= 0.3          │      C      │ 자율주행 (MALE Case 1)   │
├──────┼────────────────────────────────┼─────────────┼──────────────────────────┤
│  4   │ A + L >= 0.9                   │      C      │ 결합 고요구사항          │
├──────┼────────────────────────────────┼─────────────┼──────────────────────────┤
│  5   │ L >= 0.5                       │      B      │ 실시간 로봇 (MALE Case 2)│
├──────┼────────────────────────────────┼─────────────┼──────────────────────────┤
│  6   │ E >= 0.5 AND L < 0.5           │      A      │ IoT 센서 (MALE Case 3)   │
├──────┼────────────────────────────────┼─────────────┼──────────────────────────┤
│  7   │ Default                        │      A      │ Best-Effort RT           │
└──────┴────────────────────────────────┴─────────────┴──────────────────────────┘


Criticality Levels:

┌─────────────────────┬─────────────────────────────────────────────────────────┐
│ C (Safety-Critical) │ 데드라인 미스 = 재앙적 결과                             │
│                     │ 예: 충돌 회피, 로봇 팔 제어                             │
├─────────────────────┼─────────────────────────────────────────────────────────┤
│ B (Mission-Critical)│ 데드라인 미스 = 성능 저하                               │
│                     │ 예: SLAM, 센서 융합                                     │
├─────────────────────┼─────────────────────────────────────────────────────────┤
│ A (Best-Effort RT)  │ 데드라인 미스 = 허용 가능                               │
│                     │ 예: 로깅, 모니터링                                      │
└─────────────────────┴─────────────────────────────────────────────────────────┘
```

## 컴포넌트 역할 요약

### Central Cluster 컴포넌트

| 컴포넌트 | 역할 |
|----------|------|
| **Karmada Control Plane** | 멀티 클러스터 관리, 워크로드 전파 정책 |
| **MALE-Operator** | MaleWorkload 감시, ALE 기반 MC Criticality 재정의 |
| **MALE-Advisor** | ALE 점수 계산, 정책 결정 (Python gRPC) |
| **MALE-Profiler** | 워크로드 프로파일링, 분석 |
| **Metric-Collector** | 메트릭 수집 (RabbitMQ + InfluxDB) |

### Edge Cluster 컴포넌트

| 컴포넌트 | 역할 |
|----------|------|
| **Karmada Agent** | Central에서 Work 리소스 수신, 로컬 리소스 생성 |
| **SDI-Scheduler** | MC-aware 실시간 스케줄링, InfluxDB 메트릭 기반 결정 |
| **RTContainer CRD** | 실시간 워크로드 정의 (criticality, period, wcet, deadline) |
